What is a virtual environment?

  • A Python virtual environment is an isolated environment on your computer where you can run and test your Python projects.
  • It allows you to manage project-specific dependencies without interfering with other projects or the original Python installation.
  • Think of a virtual environment as a separate container for each Python project.

    • The use of virtual environments is important because:

  • Avoids package version conflicts between projects
  • Makes projects more portable and reproducible
  • Keeps the Python installation on your system clean
  • Allows testing with different versions of Python
    • # Create a virtual environment named 'myproject'
python -m venv myproject

# Activate the virtual environment
# On macOS/Linux
source myproject/bin/activate

    # On Windows
myproject\Scripts\activate

    Core Concepts

    • Selenium: Selenium is an open source framework used to automate web browser testing. It allows developers and testers to simulate user interaction with a browser, such as clicking buttons, filling in forms and navigating between pages. This is done through scripts that control the browser, allowing the automation of functional and regression tests in web applications
    • Pytest: Pytest is a popular and powerful testing framework for Python, used to write and execute software tests. It is known for its simplicity, flexibility and ease of use, making it a popular choice for both simple unit tests and complex functional tests.

    Install Dependencies:

    2. Browser Driver Setup

    Download the correct WebDriver for your browser.

  • WebDriver: The component that acts as a bridge between your test script (e.g., Selenium) and the web browser. Each browser has its own specific WebDriver (like Chromedriver for Chrome or Geckodriver for Firefox) that translates your script's commands into actions the browser can understand.

    • Selenium requires a WebDriver to interface with your chosen browser. Download the driver that matches your browser and version.

    • ChromeDriver for Google Chrome.
    • GeckoDriver for Mozilla Firefox.

    After downloading, place the executable in a directory that is part of your system's PATH (e.g., /usr/local/bin on macOS/Linux or a specific folder on Windows that you add to Environment Variables).

    sudo mv chromedriver /usr/local/bin/

    4. First Test Script

    Write a simple test to open a website and verify its title.

    Create a file named test_demo.py. Pytest automatically discovers files prefixed with test_.

    # test_demo.py
from selenium import webdriver

def test_verify_demoqa_title():
    driver = webdriver.Chrome()
    driver.get("https://demoqa.com")
    assert "DEMOQA" in driver.title
    driver.quit()

    To run this test, simply execute pytest in your terminal.

    5. Using Pytest Fixtures (conftest.py)

    Create a fixture to manage the WebDriver lifecycle.

    Fixtures help set up and tear down resources for your tests. Create a conftest.py file in your project's root directory.

    # conftest.py
import pytest
from selenium import webdriver

@pytest.fixture
def driver():
    # Setup: initialize the WebDriver
    driver_instance = webdriver.Chrome()
    yield driver_instance
    # Teardown: close the WebDriver
    driver_instance.quit()

    Refactor the test to use the fixture:

    Now, update test_demo.py to use the `driver` fixture.

    # test_demo.py
def test_verify_demoqa_title(driver):
    driver.get("https://demoqa.com")
    assert "DEMOQA" in driver.title
